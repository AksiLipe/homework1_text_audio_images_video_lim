## Отчёт по заданию: приложения с готовыми ML-моделями

### Оборудование и софт
- CPU/GPU: (вставь сюда: например, GTX 1080 Ti, Ryzen 5 2600)
- RAM: (вставь)
- Python: (вставь версию)
- Виртуальное окружение: `.venv`

### Что сделано
Реализованы 4 задачи под разные типы контента + отдельный скрипт для локального LLM:
1) `text/` — анализ тональности текста (классификация) + FastAPI API-контур
2) `audio/` — классификация аудио-событий через TensorFlow Hub (YAMNet)
3) `images/` — детекция объектов через YOLOv5 (PyTorch)
4) `video/` — классификация жестов по кадрам через SigLIP (majority vote)
5) `llm/` — локальная генерация текста

Все скрипты пишут результаты в папку `results/`.

## 1) Текст: анализ тональности

### Цель
Определить тональность входного текста.

### Библиотеки и модель
- `transformers` (Hugging Face)
- Модель по умолчанию: `tabularisai/multilingual-sentiment-analysis` (positive/negative/neutral)

### Структура датасета
Ожидается локальный формат:
`text/data/<label>/*.txt`, где `label` — папка с именем класса.

Если локально данных нет, скрипт использует fallback: `GLUE / SST-2`.

### Метрика качества
- `accuracy` (точность классификации), если есть возможность сопоставить предсказания с разметкой.

### Принцип работы
Скрипт грузит pipeline `text-classification` (Hugging Face Transformers), прогоняет тексты и сохраняет:
- `y_true`, `y_pred` и `accuracy` (если вычислилось).

Точность в локальном режиме считается только тогда, когда можно сопоставить названия папок (`text/data/<label>/...`)
с предсказанными строковыми метками модели (`positive`/`negative`/`neutral`). Сопоставление case-insensitive.

### Где смотреть результат
- `results/text_sentiment_*.json`

## 2) Аудио: классификация событий (YAMNet)

### Цель
Определить наиболее вероятные звуковые события в аудио.

### Библиотеки и модель
- `tensorflow` + `tensorflow-hub`
- Модель по умолчанию: `https://tfhub.dev/google/yamnet/1` (YAMNet, 521 классов)

### Структура датасета
Ожидается локальный формат:
- `audio/data/**/*.wav`
- опционально sidecar для качества:
  - `my_audio.wav -> my_audio.txt`
  - файл `.txt` содержит ground truth `display_name` (как в YAMNet)

### Метрика качества
- `accuracy` по top-1 предсказанию YAMNet, если рядом нашёлся ground-truth sidecar.
- Если ground-truth отсутствует — метрика не считается, сохраняются только предсказания.

### Принцип работы
Скрипт:
1) загружает YAMNet из TensorFlow Hub
2) ресемплит аудио к 16kHz и прогоняет модель
3) усредняет scores по фреймам, берёт top-k классов (и top-1 для accuracy)
4) при наличии sidecar сравнивает предсказанный `display_name` с ground truth

### Где смотреть результат
- `results/audio_yamnet_*.json`

## 3) Изображения: детекция объектов (YOLOv5)

### Цель
Оценить, какие объекты YOLOv5 видит на картинке, и сравнить это с “истинным” классом из папки.

### Библиотеки и модель
- PyTorch + `torch.hub`
- Модель по умолчанию: `ultralytics/yolov5` (`yolov5s`)

### Структура датасета
Ожидается локальный формат:
`images/data/<label>/*.(jpg|jpeg|png|bmp)`

Для подсчёта accuracy `<label>` должен соответствовать одному из COCO классов, которые выдаёт YOLOv5 (например `person`, `car`).

### Метрика качества
- `accuracy`: сравниваем папочный `<label>` с предсказанным top-1 классом с максимальной confidence (если confidence >= `conf-thres`).

### Принцип работы
1) YOLOv5 запускается на изображении
2) выбирается detection с максимальной confidence
3) pred_label сравнивается с `<label>` из пути

### Где смотреть результат
- `results/images_yolov5_*.json`

## 4) Видео: классификация жестов по кадрам (SigLIP)

### Цель
Определить, какой жест представлен в видеоролике (по выборке кадров).

### Библиотеки и модель
- OpenCV для извлечения кадров
- `transformers` + PyTorch
- Модель: `prithivMLmods/Hand-Gesture-19` (SigLIP)

### Структура датасета
Ожидается локальный формат:
`video/data/<label>/*.mp4` (или mkv/avi/mov/webm)

`<label>` берётся из папки и используется как “ground truth” для accuracy.

Если локально нет видео, можно запустить fallback-ролик (опционально скачивается по URL).

### Метрика качества
- `accuracy` по видеозаписям, когда есть ground truth (локальная структура с папками).
- В fallback-режиме метрика может быть недоступна (нет истинных классов).

### Принцип работы
1) равномерно берём `num_frames` кадров
2) для каждого кадра делаем классификацию жестов SigLIP-моделью
3) итог по видео — majority vote по предсказанным жестам

### Где смотреть результат
- `results/video_gesture_*.json`

## 5) LLM: локальная генерация

### Цель
Показать работу локальной генеративной модели.

### Библиотеки и модель
- `transformers`
- По умолчанию: `TinyLlama/TinyLlama-1.1B-Chat-v1.0`

### Метрика качества
Для LLM в этом задании обычно достаточно:
- примера промпта
- полученного ответа

В скрипте дополнительно сохраняются `seconds` (время инференса) и `device` (cpu/cuda).

### Где смотреть результат
- `results/llm_*.json`

## API-обёртка (требование задания)
В качестве API-контурa реализован `text/app.py` на FastAPI:
- простой вход: текст (JSON: `{"text": "..."}`)
- простой выход: label и score от модели.

Аналогично можно сделать API и для изображений/видео, но там сложнее обработка файлов и задержка на извлечение кадров.
